1     # Spotify Review Discovery Engine — Phase-wise Implementation Plan 

2      

3     ## 0. Purpose 

4      

5     Build a standalone AI-powered review intelligence system for Spotify that: 

6     - ingests review/discussion data from App Store, Play Store, Reddit, and similar public sources 

7     - stores raw + processed data in a free-tier database 

8     - cleans, deduplicates, tags, segments, and summarizes review data 

9     - exposes the workflow through a public Streamlit app 

10     - produces insight-ready outputs for later product work 

11      

12     This system is independent from the Spotify Vibe MVP. 

13      

14     --- 

15      

16     ## 1. Target Outcome 

17      

18     Deliver a public Streamlit app with: 

19     1. source ingestion controls 

20     2. review explorer 

21     3. theme and segment dashboards 

22     4. insight summaries 

23     5. semantic query interface ("Ask the reviews") 

24      

25     The app should be deployable from GitHub via Streamlit Community Cloud. Streamlit Community Cloud supports deploying apps directly from GitHub repositories and automatically updates apps from repository changes【5065†】【5067†】.

50     - Python + pandas

52     - optional LLM API for theme extraction / summarization

54     ### Why this stack 

55     - Streamlit Community Cloud is free and designed for public data apps that deploy from GitHub【5067†】. 

56     - Supabase Free provides a Postgres database with 500 MB DB size and 1 GB file storage, but free projects pause after one week of inactivity【5070†】. 

57     - `google-play-scraper` supports Python review crawling with pagination and review extraction without external dependencies【5077†】.

6. Data Ingestion Architecture 

6.1 Sources 

A. Google Play Reviews 

Use Python package google-play-scraper. The package provides reviews() and reviews_all() methods, supports review pagination, and uses country/lang parameters【5077†】. 

B. App Store Reviews 

Use Apple review feed / API-compatible retrieval approaches. Apple explicitly documents App Store ratings and written reviews as product feedback signals and confirms reviews remain visible independently from rating resets【5084†】. 

C. Reddit Discussions 

Use public Reddit endpoints or lightweight scraping for relevant public threads. 

D. Similar Sources 

Implement a manual CSV import pathway for: 

public forums 

social conversation exports 

any source that is easier to sample manually than automate initially

Normalization rules: 

body is required 

timestamps converted to ISO 8601 UTC 

source names are controlled values 

compute raw_hash = sha256(normalized text + source + external_id fallback)

9.3 src/process/dedupe.py 

Three-stage dedupe: 

exact hash match 

normalized text similarity 

semantic similarity fallback 

Implementation: 

exact duplicate: same raw_hash 

near duplicate: 

TF-IDF cosine similarity > threshold 

or normalized Levenshtein similarity > threshold 

semantic duplicate: 

optional embedding cosine similarity threshold 

Write: 

is_duplicate 

duplicate_of

18. MVP Acceptance Criteria 

The Review Discovery Engine is complete when: 

reviews from at least 3 sources can be ingested 

all records are normalized into one DB schema 

duplicates are handled 

themes are assigned 

user segments are inferred 

insights are generated 

a Streamlit app exposes the end-to-end workflow 

the app is publicly accessible via link 

 

19. Non-Goals 

Do not implement: 

private user research ingestion 

recommendation generation for end users 

actual Spotify account integration 

production-grade large-scale scraping infrastructure 

This system is a review analysis workflow, not the end-user MVP. 

 

20. Final Handoff to Agent 

Build this system as a self-contained review intelligence app. Prioritize: 

stable ingestion 

reliable schema 

explainable processing 

clean Streamlit workflow 

public deployment 

Do not over-engineer the LLM layer first. The critical path is: ingestion → storage → cleaning → themes → segments → insights → public app.
